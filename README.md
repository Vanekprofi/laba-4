## Задача 1. Обобщенная коробка (Box)

### Текст задачи
Создайте сущность `Коробка`, которая обладает следующими характеристиками:
1. Может хранить один произвольный объект в один момент времени.
2. Объект можно получить и разместить на хранение в любой момент времени.
3. Если объект забирают из коробки – ссылку на этот объект необходимо обнулить.
4. Если объект кладут в коробку, но она не пуста – необходимо выкинуть исключение.
5. Имеет метод проверки на заполненность.
6. Методы класса должны работать с тем типом данных, который был указан во время создания объекта.

**Требования к демонстрации:**
- Создать Коробку, хранящую целочисленное значение.
- Разместить туда число `3`.
- Передать Коробку в метод, извлечь значение и вывести его на экран.

### Алгоритм решения
- Реализовать класс `Box<T>`, содержащий поле для хранения объекта `private T value`.
- Реализовать методы:
  - `void put(T item)` — с проверкой на пустоту.
  - `T get()` — с обнулением ссылки после извлечения.
  - `boolean isEmpty()` — проверка состояния.

## Задача 2. Хранилище без null (Storage)

### Текст задачи
Создайте сущность `Хранилище`, которая обладает следующими характеристиками:
1. Может хранить один произвольный объект в один момент времени.
2. Хранилище неизменяемо (Immutable).
3. Объект кладется в Хранилище при его создании.
4. В качестве объекта может быть сохранено также и значение `null`.
5. Хранилище может вернуть ссылку на Объект.
6. Если вместо объекта хранится `null`, необходимо вернуть какое-либо альтернативное значение.
7. Метод получения значения должен работать с тем типом данных, который был указан во время создания объекта.

**Требования к демонстрации:**
1. Создать Хранилище чисел, положить `null`. Извлечь (альтернатива `0`).
2. Создать Хранилище чисел, положить `99`. Извлечь (альтернатива `-1`).
3. Создать Хранилище строк, положить `null`. Извлечь (альтернатива `"default"`).
4. Создать Хранилище строк, положить `"hello"`. Извлечь (альтернатива `"world"`).

### Алгоритм решения
- Реализовать класс `Storage<T>` с финальным полем `value`.
- Реализовать метод `T getValue(T alternative)`, который возвращает `value`, если оно не `null`, иначе возвращает `alternative`.

## Задача 3. Начало отсчета (Point3D & Box)

### Текст задачи
Создайте метод, принимающий `Коробку` из Задачи 1, и кладет в неё трехмерную точку координат (`Point3D` из задачи 2.1.5) с произвольными значениями. 
Метод должен позволять передавать Коробку с более чем одним видом параметризации (использовать нижнюю границу типа).

### Алгоритм решения
- Реализовать метод, который кладёт трёхмерную точку в коробку.
- Сигнатура метода должна принимать `Box<? super Point3D>`, чтобы обеспечить гибкость типов.
